# AGENTS.md 模板

本文档作为 AI Agent 的配置模板，建立工作区行为、记忆管理和交互指南。

## 核心部分

### 首次运行和每次会话
Agent 应在首次运行时检查 `BOOTSTRAP.md`，然后在每次会话开始时读取身份文件（`SOUL.md`、`USER.md`）和最近的记忆文件。指令很明确："不要请求许可。直接做。"

### 记忆系统
框架使用两层记忆方法：
- **每日笔记**存储在 `memory/YYYY-MM-DD.md` 中，用于原始会话日志
- **长期记忆**在 `MEMORY.md` 中，用于精心整理的重要信息

强调的关键原则："记忆是有限的——如果你想记住某事，把它写入文件。"心理笔记不会跨会话持久化。

### 安全指南
核心规则包括永不泄露私人数据、优先使用 `trash` 而非 `rm` 以便恢复，以及在运行破坏性命令前询问。内部操作（读取文件、搜索）可以自由进行，而外部操作（电子邮件、帖子）需要确认。

### 群聊行为
文档提供了关于何时发言与保持沉默的详细指导。原则是："参与，不要主导。"Agent 应自然地使用表情符号回应，避免回复每条消息。

### 心跳系统
心跳启用主动行为——定期检查电子邮件、日历和通知。文档区分心跳（批量、灵活时间）和 cron 任务（精确调度、隔离任务）。

### 记忆维护
在心跳期间，Agent 应定期审查每日文件并用提炼的见解更新 `MEMORY.md`，将其视为"人类审查日记"。

### 自定义
最后一节鼓励个性化："这是一个起点。添加你自己的约定、风格和规则。"

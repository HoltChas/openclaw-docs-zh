快速入门

目标：从零开始，用最合理的默认配置，尽快完成首次正常对话。

最快的聊天方式：打开控制界面（无需配置渠道）。运行 openclaw dashboard 在浏览器中聊天，或在网关主机上打开 http://127.0.0.1:18789/。

相关文档：控制台 和 控制界面。

推荐路径

使用 CLI 初始化向导（openclaw onboard）。它会设置：

  • 模型/认证（推荐 OAuth）
  • 网关设置
  • 渠道（WhatsApp/Telegram/Discord/Mattermost 等）
  • 配对默认设置（安全的私信）
  • 工作区引导 + 技能
  • 可选的后台服务

如需深入了解参考页面，请跳转到：向导、设置、配对、安全。

沙盒说明：agents.defaults.sandbox.mode: "non-main" 使用 session.mainKey（默认为 "main"），因此群组/渠道会话会被沙盒化。如果希望主代理始终在主机上运行，请设置明确的每代理覆盖：

{
  "routing": {
    "agents": {
      "main": {
        "workspace": "~/.openclaw/workspace",
        "sandbox": { "mode": "off" }
      }
    }
  }
}

0) 前置要求

  • Node >= 22
  • pnpm（可选；如果从源码构建则推荐）
  • 推荐：Brave Search API 密钥用于网页搜索。最简单的方式：openclaw configure --section web（存储在 tools.web.search.apiKey）。详见 Web 工具。

macOS：如果计划构建应用，请安装 Xcode / CLT。仅使用 CLI + 网关时，Node 就足够了。

Windows：使用 WSL2（推荐 Ubuntu）。强烈推荐 WSL2；原生 Windows 未经测试，问题更多，工具兼容性较差。先安装 WSL2，然后在 WSL 中运行 Linux 步骤。详见 Windows (WSL2)。

1) 安装 CLI（推荐）

curl -fsSL https://openclaw.bot/install.sh | bash

Windows (PowerShell)：

iwr -useb https://openclaw.ai/install.ps1 | iex

替代方案（全局安装）：

npm install -g openclaw@latest
# 或: pnpm add -g openclaw@latest

2) 运行初始化向导（并安装服务）

openclaw onboard --install-daemon

你需要选择：

  • 本地 vs 远程网关
  • 认证：OpenAI Code (Codex) 订阅（OAuth）或 API 密钥。对于 Anthropic，我们推荐 API 密钥；也支持 claude setup-token。
  • 提供商：WhatsApp 二维码登录、Telegram/Discord 机器人令牌、Mattermost 插件令牌等
  • 守护进程：后台安装（launchd/systemd；WSL2 使用 systemd）
  • 运行时：Node（推荐；WhatsApp/Telegram 必需）。不推荐使用 Bun。
  • 网关令牌：向导默认生成一个（即使在本地），并存储在 gateway.auth.token 中

向导文档：向导

认证：存储位置（重要）

  • 推荐的 Anthropic 路径：设置 API 密钥（向导可以为你存储以服务使用）。也支持 claude setup-token 如果你想重用 Claude Code 凭证。
  • OAuth 凭证（旧版导入）：~/.openclaw/credentials/oauth.json
  • 认证配置文件（OAuth + API 密钥）：~/.openclaw/agents//agent/auth-profiles.json

无头/服务器提示：先在普通机器上做 OAuth，然后将 oauth.json 复制到网关主机。

3) 启动网关

如果你在初始化过程中安装了服务，网关应该已经在运行：

openclaw gateway status

手动运行（前台）：

openclaw gateway --port 18789 --verbose

控制台（本地）：http://127.0.0.1:18789/

如果配置了令牌，将其粘贴到控制界面设置中（存储为 connect.params.auth.token）。

⚠️ Bun 警告（WhatsApp + Telegram）：Bun 对这些渠道有已知问题。如果使用 WhatsApp 或 Telegram，请使用 Node 运行网关。

3.5) 快速验证（2 分钟）

openclaw status
openclaw health
openclaw security audit --deep

4) 配对并连接你的第一个聊天界面

WhatsApp（二维码登录）

openclaw channels login

通过 WhatsApp → 设置 → 关联设备扫描。

WhatsApp 文档：WhatsApp

Telegram / Discord / 其他

向导可以为你写入令牌/配置。如果偏好手动配置，从以下开始：

  • Telegram：Telegram
  • Discord：Discord
  • Mattermost（插件）：Mattermost

Telegram 私信提示：你的第一个私信会返回一个配对码。批准它（见下一步）否则机器人不会回复。

5) 私信安全（配对批准）

默认策略：未知私信会收到一个短代码，消息在批准前不会被处理。如果你的第一条私信没有回复，请批准配对：

openclaw pairing list whatsapp
openclaw pairing approve whatsapp <CODE>

配对文档：配对

从源码运行（开发）

如果你正在修改 OpenClaw 本身，请从源码运行：

git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build  # 首次运行时自动安装 UI 依赖
pnpm build
openclaw onboard --install-daemon

如果还没有全局安装，请通过 pnpm openclaw ... 从仓库运行初始化步骤。

pnpm build 也会打包 A2UI 资源；如果只需要运行该步骤，使用 pnpm canvas:a2ui:bundle。

网关（从此仓库）：

node openclaw.mjs gateway --port 18789 --verbose

7) 端到端验证

在新终端中，发送测试消息：

openclaw message send --target +15555550123 --message "Hello from OpenClaw"

如果 openclaw health 显示 "no auth configured"，请返回向导设置 OAuth/密钥认证 — 没有认证代理将无法回复。

提示：openclaw status --all 是最佳的可粘贴、只读调试报告。

健康探测：openclaw health（或 openclaw status --deep）会向运行中的网关请求健康快照。
